Классификация клиентов совершивших или не совершивших подписку на банковский депозит.

Дана база данных клиентов, с которыми проводилась коммуникации в различном формате.
Подробное описание признаков ниже. Есть данные о клиенте, о взаимодейтсии и о результатах.

1. Файл preparing
- разбирается база клиентов. 
- создаём пайплайн обработки данных
- определяем важность признаков
- по результатам оценки важности признаков выбираем 5 признаков: 
month - месяц последней коммуникации
duration - длительность коммуникации
pdays - число дней с последнего контакта
euribor3m - банковский показатель
nr_employed - номер сотрудника
- разделяем и сохраняем тренировочные и тестовые выборки
- обучаем модель с отсавшимися признаками
- сохраняем модель

2. Файл pred_test
- проверяем сохранённую модель

3. run_server
-делаем конфигурацию принимаемого запроса
- запускаем сервер

4. model_interract
- формируем запрос
- проверяем работу на 150 элементах выборки
- получили roc_auc 0,912, но в выборке значительный дисбаланс, поэтому пробуем pres/recall
- по метрике f_score = 0,667
